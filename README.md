# portal-home-hub

## 项目简介

portal-home-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为一组静态页面的托管与分发使用。每个页面都是独立的，彼此之间没有依赖关系，方便快速浏览、分享或嵌入到其他环境中。

## 目录结构

```
portal-home-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 共享资源文件（如样式、脚本、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- **pages/**：每个页面以单独文件夹或文件形式存放，命名清晰，便于查找。
- **assets/**：存放页面可能引用的公共资源，如 CSS、JavaScript、字体、图标等。各页面也可以自行维护私有资源。

## 页面归档说明

本仓库中的每个 HTML 页面均经过整理和归档，以确保其结构清晰、内容完整。归档规则如下：

- 每个页面独立成文，不依赖外部服务（除非明确声明）。
- 页面文件名通常反映其主题或功能，例如 `example-page.html`。
- 资源引用尽量使用相对路径，方便本地或静态托管服务（如 GitHub Pages）直接使用。

## 维护说明

- **添加新页面**：在 `pages/` 下创建新的 HTML 文件或文件夹，并确保所有引用的资源文件位于 `assets/` 或页面自身目录中。
- **更新页面**：直接修改对应文件，注意保持向后兼容，避免破坏现有引用。
- **删除页面**：移除对应文件或文件夹，并检查是否有其他页面引用该资源。
- **提交规范**：建议在提交信息中简要说明变更内容，例如“新增：xxx 页面”或“修复：xxx 页面样式问题”。

## 使用方式

你可以通过以下方式使用本仓库中的页面：

1. **直接浏览**：克隆仓库到本地，打开 `pages/` 下的 HTML 文件即可查看。
2. **静态托管**：将仓库部署到支持静态网站的托管服务（如 GitHub Pages），访问对应 URL 即可在线浏览。
3. **嵌入引用**：将页面文件或资源链接嵌入到其他项目中，需注意跨域及路径问题。
